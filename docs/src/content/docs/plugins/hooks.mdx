Hooks let plugins run code in response to CMS events. Every hook handler receives two arguments: an event object and the plugin context (`ctx`). Hooks are declared at definition time inside `definePlugin()`, not registered dynamically.

```typescript
hooks: {
	"content:afterSave": async (event, ctx) => {
		ctx.log.info(`Saved ${event.collection}/${event.content.id}`);
	},
}
```

## Hook Configuration

Hooks can be a bare function or a config object with priority, timeout, and error handling:

<Tabs>
	<TabItem label="Simple">
```typescript
hooks: {
	"content:afterSave": async (event, ctx) => {
		ctx.log.info("Content saved");
	},
}
```
	</TabItem>
	<TabItem label="With Options">
```typescript
hooks: {
	"content:afterSave": {
		priority: 200,
		timeout: 10000,
		errorPolicy: "continue",
		handler: async (event, ctx) => {
			ctx.log.info("Content saved");
		},
	},
}
```
	</TabItem>
</Tabs>

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `priority` | `number` | `100` | Execution order. Lower numbers run first. |
| `timeout` | `number` | `5000` | Max execution time in milliseconds. |
| `dependencies` | `string[]` | `[]` | Plugin IDs that must run before this hook. |
| `errorPolicy` | `"abort" \| "continue"` | `"abort"` | Whether to stop the hook pipeline on error. |

## Content Hooks

### `content:beforeSave`

Fires before content is written to the database. Return modified content to transform it, or throw to cancel the save.

**Event data:**

```typescript
{
	content: Record<string, unknown>;  // The content being saved
	collection: string;                // Collection slug (e.g., "posts")
	isNew: boolean;                    // true = create, false = update
}
```

**Returns:** `Record<string, unknown> | void`

**Example -- slug normalization:**

```typescript
"content:beforeSave": async (event, ctx) => {
	const { content, collection, isNew } = event;

	if (content.slug) {
		content.slug = content.slug.toLowerCase().replace(/\s+/g, "-");
	}

	return content;
},
```

### `content:afterSave`

Fires after content is successfully saved. Use for side effects: logging, notifications, external syncs, validation warnings. Cannot modify the saved content.

**Event data:**

```typescript
{
	content: Record<string, unknown>;  // The saved content (includes id, timestamps)
	collection: string;
	isNew: boolean;
}
```

**Returns:** `void`

**Real example -- the SEO plugin validates meta tags after every save:**

```typescript
"content:afterSave": async (event, _ctx) => {
	const { data } = event.content;
	const seo = data.seo as
		| { title?: string; description?: string; ogImage?: string }
		| undefined;

	if (!seo) return;

	const warnings: string[] = [];

	// Google truncates titles at ~60 characters
	if (seo.title && seo.title.length > 60) {
		warnings.push(
			`SEO title is ${seo.title.length} chars (recommended: under 60)`,
		);
	}

	// Google truncates descriptions at ~160 characters
	if (seo.description && seo.description.length > 160) {
		warnings.push(
			`SEO description is ${seo.description.length} chars (recommended: under 160)`,
		);
	}

	if (!seo.ogImage) {
		warnings.push("No Open Graph image set. Social shares will use a default image.");
	}

	if (warnings.length > 0) {
		console.log(`[seo] Warnings for "${data.title || "Untitled"}":`, warnings);
	}
},
```

**Real example -- the audit-log plugin records every content change:**

```typescript
"content:afterSave": {
	priority: 200, // Run after other plugins
	timeout: 2000,
	handler: async (event, ctx) => {
		const { content, collection, isNew } = event;

		const entry = {
			timestamp: new Date().toISOString(),
			action: isNew ? "create" : "update",
			collection,
			resourceId: content.id as string,
		};

		const entryId = `${Date.now()}-${content.id}`;
		await ctx.storage.entries!.put(entryId, entry);

		ctx.log.info(`Logged ${entry.action} on ${collection}/${content.id}`);
	},
},
```

### `content:afterDelete`

Fires after content is permanently deleted. Use for cleanup: removing related plugin data, notifying external systems, audit logging.

**Event data:**

```typescript
{
	id: string;           // ID of the deleted content
	collection: string;   // Collection slug
}
```

**Returns:** `void`

**Example -- clean up plugin data after deletion:**

```typescript
"content:afterDelete": async (event, ctx) => {
	const { id, collection } = event;

	// Remove cached data for this content
	await ctx.storage.cache!.delete(`${collection}:${id}`);

	ctx.log.info(`Cleaned up data for deleted ${collection}/${id}`);
},
```

## Media Hooks

### `media:beforeUpload`

Fires before a file is uploaded. Return modified file info to rename or reclassify, or throw to reject the upload.

**Event data:**

```typescript
{
	file: {
		name: string;    // Original filename
		type: string;    // MIME type (e.g., "image/png")
		size: number;    // Size in bytes
	}
}
```

**Returns:** `{ name: string; type: string; size: number } | void`

**Example -- enforce image-only uploads with a 10MB limit:**

```typescript
"media:beforeUpload": async (event, ctx) => {
	const { file } = event;

	if (!file.type.startsWith("image/")) {
		throw new Error("Only images are allowed");
	}

	if (file.size > 10 * 1024 * 1024) {
		throw new Error("File too large (max 10MB)");
	}

	// Prefix filename with timestamp
	return { ...file, name: `${Date.now()}-${file.name}` };
},
```

### `media:afterUpload`

Fires after a file is successfully uploaded and stored.

**Event data:**

```typescript
{
	media: {
		id: string;
		filename: string;
		mimeType: string;
		size: number | null;
		url: string;
		createdAt: string;
	}
}
```

**Returns:** `void`

**Example -- log upload details:**

```typescript
"media:afterUpload": async (event, ctx) => {
	const { media } = event;

	ctx.log.info(`Uploaded ${media.filename}`, {
		id: media.id,
		size: media.size,
		mimeType: media.mimeType,
	});
},
```

### `media:afterDelete`

Fires after a media item is deleted from storage.

**Event data:**

```typescript
{
	id: string;           // ID of the deleted media item
	filename: string;
}
```

**Returns:** `void`

**Example:**

```typescript
"media:afterDelete": async (event, ctx) => {
	ctx.log.info(`Deleted media ${event.filename} (${event.id})`);
},
```

## Lifecycle Hooks

These hooks run during plugin installation and state changes.

### `plugin:install`

Fires once when the plugin is first added to a site. Seed default data here.

```typescript
"plugin:install": async (_event, ctx) => {
	await ctx.kv.set("settings:enabled", true);
	ctx.log.info("Plugin installed with default settings");
},
```

### `plugin:activate` / `plugin:deactivate`

Fire when the plugin is enabled or disabled from the admin panel.

```typescript
"plugin:activate": async (_event, ctx) => {
	ctx.log.info("Plugin activated");
},
"plugin:deactivate": async (_event, ctx) => {
	ctx.log.info("Plugin deactivated — pausing background work");
},
```

### `plugin:uninstall`

Fires when the plugin is removed. The event includes `deleteData: boolean` indicating whether the user chose to wipe plugin data.

```typescript
"plugin:uninstall": async (event, ctx) => {
	if (event.deleteData) {
		const result = await ctx.storage.entries!.query({ limit: 1000 });
		await ctx.storage.entries!.deleteMany(result.items.map((i) => i.id));
		ctx.log.info("Plugin data deleted");
	}
},
```

<Aside type="caution">
	Default to preserving data in `plugin:uninstall`. Users may reinstall. Only delete when
	`event.deleteData` is `true`.
</Aside>

## Public Page Hooks

### `page:metadata`

Contributes structured metadata to rendered pages -- meta tags, Open Graph properties, canonical links, and JSON-LD. Works in both trusted and sandboxed modes.

```typescript
"page:metadata": async (event, ctx) => {
	if (event.page.kind !== "content") return null;

	return {
		kind: "jsonld",
		graph: {
			"@context": "https://schema.org",
			"@type": "BlogPosting",
			headline: event.page.title,
			description: event.page.description,
		},
	};
},
```

### `page:fragments`

Injects raw HTML, scripts, or styles into page insertion points. **Trusted plugins only** -- sandboxed plugins cannot use this hook.

```typescript
"page:fragments": async (event, ctx) => {
	return {
		kind: "external-script",
		placement: "head",
		src: "https://www.googletagmanager.com/gtm.js?id=GTM-XXXXX",
		async: true,
	};
},
```

Placements: `"head"`, `"body:start"`, `"body:end"`.

<Aside type="caution">
	`page:fragments` output runs as first-party code in the browser. Sandbox isolation does not
	extend to browser-executed scripts. Use `page:metadata` for sandbox-safe contributions.
</Aside>

## Hook Execution Order

1. Hooks with lower `priority` values run first (default is `100`).
2. Hooks with equal priority run in plugin registration order.
3. Hooks with `dependencies` wait for those plugins to complete first.

```typescript
// Plugin A: priority 50 -- runs first
"content:afterSave": { priority: 50, handler: async () => {} }

// Plugin B: default priority 100 -- runs second
"content:afterSave": async () => {}

// Plugin C: depends on A -- waits for A regardless of priority
"content:afterSave": { dependencies: ["plugin-a"], handler: async () => {} }
```

## Error Handling

- **`errorPolicy: "abort"`** (default) -- Pipeline stops, original operation may fail.
- **`errorPolicy: "continue"`** -- Error is logged, remaining hooks still run.

Use `"continue"` for non-critical work like analytics, notifications, or external syncs where failure should not block the user's action.

## Quick Reference

| Hook | Fires when | Event data | Can modify? |
| --- | --- | --- | --- |
| `content:beforeSave` | Before content write | `{ content, collection, isNew }` | Return modified content |
| `content:afterSave` | After content write | `{ content, collection, isNew }` | No |
| `content:afterDelete` | After content delete | `{ id, collection }` | No |
| `media:beforeUpload` | Before file upload | `{ file: { name, type, size } }` | Return modified file info |
| `media:afterUpload` | After file upload | `{ media: { id, filename, ... } }` | No |
| `media:afterDelete` | After media delete | `{ id, filename }` | No |
| `plugin:install` | First installation | `{}` | No |
| `plugin:activate` | Plugin enabled | `{}` | No |
| `plugin:deactivate` | Plugin disabled | `{}` | No |
| `plugin:uninstall` | Plugin removed | `{ deleteData }` | No |
| `page:metadata` | Page render | `{ page }` | Return contributions |
| `page:fragments` | Page render (trusted) | `{ page }` | Return contributions |
